# Label Conventions

Standard labels for issues, PRs, and task tracking.

## Type Labels
| Label | Color | Description |
|-------|-------|-------------|
| `bug` | #d73a4a | Something isn't working |
| `feat` | #0075ca | New feature or enhancement |
| `docs` | #0052cc | Documentation changes |
| `chore` | #666666 | Maintenance, dependencies, configs |
| `refactor` | #1d76db | Code improvement without behavior change |
| `test` | #bfd4f2 | Test additions or fixes |
| `research` | #5319e7 | Research and exploration tasks |

## Priority Labels
| Label | Color | Description |
|-------|-------|-------------|
| `priority:critical` | #b60205 | Must fix immediately |
| `priority:high` | #d93f0b | Important, do soon |
| `priority:medium` | #fbca04 | Normal priority |
| `priority:low` | #0e8a16 | Nice to have |

## Status Labels
| Label | Color | Description |
|-------|-------|-------------|
| `status:blocked` | #000000 | Waiting on something |
| `status:in-progress` | #7057ff | Currently being worked on |
| `status:review` | #006b75 | Ready for review |

## Component Labels
| Label | Color | Description |
|-------|-------|-------------|
| `component:pipeline` | #c2e0c6 | Core extraction pipeline |
| `component:chunking` | #bfdadc | Document chunking |
| `component:agents` | #d4c5f9 | AI agent system |
| `component:schema` | #fef2c0 | Node/Edge schema |

## Milestone Labels
| Label | Color | Description |
|-------|-------|-------------|
| `m1` | #c5def5 | Milestone 1 - MVP Core Pipeline |
| `m2` | #bfdadc | Milestone 2 |
| `m3` | #d4c5f9 | Milestone 3 |

## Usage

When creating issues:
1. Always add one Type label
2. Add Priority label if not obvious
3. Add Component label for code-related issues
4. Add Milestone label if part of a milestone
5. Add Status label once work begins
